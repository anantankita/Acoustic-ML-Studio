Developer Insights & Project Architecture Guide (Recho Review)

Requirements Mapping

  • PyTorch Design & Training

    Implemented in src/train.py and app/model.py. Features custom dataset class, data loader, SpecAugment, and CRNN architecture (Conv2D + GRU + Linear).

  • DSP Preprocessing & Denoising

    Spectral Gating noise reduction filter implemented inside app/audio.py using Short-Time Fourier Transform (STFT) masking. Computes Mel-spectrograms and MFCC features.

  • Model Optimization & Quantization

    Dynamic INT8 Quantization implemented in src/optimize.py. Converts model weights from float32 to int8, compressing size by ~72% and boosting execution speed.

  • High-Performance Deployment

    Hosted via FastAPI in app/main.py with CORS middleware, static directories, and clean API endpoints for processing and latency profiling.

Interview Talk Track

"To demonstrate end-to-end capabilities relevant to Recho's ASR and TTS workloads, I built a Speech Command Recognition Hub. I designed a hybrid Convolutional Recurrent Network (CRNN) in PyTorch to capture both spatial-spectral and temporal dynamics in audio. For preprocessing, I implemented standard log-Mel Spectrogram and MFCC pipelines, and wrote a custom Spectral Gating filter for noise reduction. Finally, to prepare the model for high-throughput, low-latency API deployment, I compiled it to ONNX format and applied post-training dynamic INT8 quantization, lowering model size from 0.54MB to 0.15MB and achieving a ~6.5x speedup in CPU latency, which is hosted on a FastAPI web service."
